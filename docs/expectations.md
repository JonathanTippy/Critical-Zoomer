# Critical Zoomer Owner Expectations

## Project Scope

Exploring the mandelbrot set.

## Alternatives

- Xaos (strongest UX, current Linux King)
- Kalles Fraktaler (strongest math)
- Superfractalthing (seamless perturbation)
- Ultra Fractal (perfect but paid)

## Why create another?

Apps which don't move as quickly and go as deep as desired get dropped instantly by 90% of users.

Alternatives are either deeply functional but awkward to use,
or nicely put together but use standard methods;
the field splits polish and depth across different programs.
UltraFractal is the strongest alternative, but its inaccessable. Not everyone has money.

No listed alternative is free, deep, and user-friendly in one app.

Critical Zoomer aims to fix this accessability gap.

## Central Expectations

Critical Zoomer must be Seamless, Deep, Tenacious, Hoarding, Fast, and Calibrated. Details follow.

### Seamless

There must be no "increase the max iteration count" wall which asks the bewildered user 
to increase this setting if they wish to continue.

There must be no "turn on perturbation" wall which asks the bewildered user to "choose a reference orbit".
It must choose reference orbits automatically, and perturbation must be on from the start.

### Deep

The user must be allowed to go as deep as he would care to go. At comfortable zooming speed, this is much deeper than one might expect. Advanced algorithms are mandatory.

### Tenacious

The app must discard the concept of a "max iteration count" and instead always 
attempt to finish its work, as long as its still visible.

### Hoarding

There must only be one possible answer per view, obviating any need to redo work.
Once done, work must be stored in a buffer, so its never re-done owing to a cosmetic settings change.

### Fast

The user must be allowed to zoom in as fast as he would care to go. 
To this aim, the app must not be stingy with zooming speed. It must zoom at the most natural feeling speed.
Extensive optimization, work saving, and lookahead will all be must.

All settings must apply instantly and not trigger a re-compute;
there must be no computational settings.

### Calibrated

The outputs must be reasonably believed to be correct and known.
